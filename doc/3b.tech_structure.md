│              BUN NATIVE SERVER / Hono 4 (Bun runtime)            │
│                                                                  │

│                                                                  │

### Bảng Mapping Tier → Công nghệ

| Tier | Công nghệ | Vai trò |
|:---|:---|:---|
| **Client** | React 19, Vite 6, TailwindCSS 4 | SPA rendering, styling |
| **Client State** | TanStack Query v5, Zustand, react-hook-form | Server cache, UI state, forms |
| **Client Auth** | better-auth/react | Auth hooks (useSession, signIn, signOut) |
| **API Gateway** | Docker Compose (Bun native server) | Hosting Hono app |
| **API Framework** | Hono 4 (Bun native) | Request handling, middleware |
| **Validation** | zod + @hono/zod-validator | Schema validation (shared FE/BE) |
| **ORM** | Drizzle ORM + bun:sqlite | Type-safe queries, migrations |
| **Database** | SQLite (local file ./data/app.db) | Data persistence |
| **Auth Service** | better-auth (self-hosted, cookie-based sessions) | User management, sessions |

> **Ghi chú runtime:** Bun là runtime duy nhất cho cả development lẫn production. Hono chạy native trên Bun server (không cần adapter). Deploy bằng Docker Compose.

## 2. Luồng Dữ liệu

### 2.1. Request/Response Tiêu biểu

```
User action (click, submit)
  → React component
    → TanStack Query (useMutation / useQuery)
      → apiClient.fetch("/api/transactions", { method: "POST", body })
        → Hono middleware chain:
            logger → cors → authMiddleware → zValidator("json", schema)
              → route handler → service function(userId, data)
                → Drizzle ORM → SQLite
                  → return rows
                → format response
              → JSON { data, meta? }
            ← 200 OK
          ← TanStack Query cache update
        ← React re-render với data mới
```

### 2.2. Auth Flow (Frontend → better-auth qua Hono)

Auth đi qua Hono backend. better-auth xử lý tại `/api/auth/**`:

```
User submit login form
  → authClient.signIn.email({ email, password })
    → POST /api/auth/sign-in (Hono → better-auth handler)
      ← Set-Cookie: better-auth session cookie
  → Mọi API request sau đó gửi cookie tự động (credentials: 'include')
  → Session hết hạn → better-auth tự động refresh via cookie
```

### 2.3. Cache Invalidation (TanStack Query)

Khi mutation thành công, invalidate các query keys liên quan:

| Mutation | Query Keys bị Invalidate |
|:---|:---|
| Tạo/sửa/xóa transaction | `["transactions"]`, `["dashboard"]`, `["reports"]` |
| Tạo/sửa/xóa category | `["categories"]`, `["transactions"]` |
| Tạo/sửa/xóa budget | `["budgets"]`, `["dashboard"]` |
| Tạo/sửa/xóa asset | `["assets"]`, `["dashboard"]` |
| Tạo/sửa/xóa wallet | `["wallets"]`, `["dashboard"]` |
| Cập nhật profile/settings | `["profile"]` |

### 3.4. API Client Pattern

```typescript
// apps/web/src/services/apiClient.ts
const API_BASE = import.meta.env.VITE_API_URL || 'http://localhost:3000'

type RequestOptions = Omit<RequestInit, 'body'> & {
  body?: unknown
}

export async function apiClient<T>(
  path: string,
  options: RequestOptions = {},
): Promise<T> {
  const response = await fetch(`${API_BASE}/api${path}`, {
    ...options,
    credentials: 'include',
    headers: {
      'Content-Type': 'application/json',
      ...options.headers,
    },
    body: options.body ? JSON.stringify(options.body) : undefined,
  })

  if (!response.ok) {
    const error = await response.json().catch(() => ({}))
    throw new Error(error.error?.message || `Request failed: ${response.status}`)
  }

  return response.json()
}
```

### 3.5. TanStack Query Hook Pattern

```typescript
// apps/web/src/hooks/useTransactions.ts
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query";
import { apiClient } from "../services/apiClient";
import type { Transaction, CreateTransactionInput } from "@pf/shared";

export function useTransactions(filters?: {
  page?: number;
  limit?: number;
  type?: "income" | "expense";
  categoryId?: string;
  dateFrom?: string;
  dateTo?: string;
}) {
  return useQuery({
    queryKey: ["transactions", filters],
    queryFn: () => {
      const params = new URLSearchParams();
      if (filters?.page) params.set("page", String(filters.page));
      if (filters?.limit) params.set("limit", String(filters.limit));
      if (filters?.type) params.set("type", filters.type);
      if (filters?.categoryId) params.set("categoryId", filters.categoryId);
      if (filters?.dateFrom) params.set("dateFrom", filters.dateFrom);
      if (filters?.dateTo) params.set("dateTo", filters.dateTo);
      const qs = params.toString();
      return apiClient<{ data: Transaction[]; meta: PaginationMeta }>(
        `/transactions${qs ? `?${qs}` : ""}`,
      );
    },
  });
}

export function useCreateTransaction() {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: (data: CreateTransactionInput) =>
      apiClient<Transaction>("/transactions", {
        method: "POST",
        body: data,
      }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["transactions"] });
      queryClient.invalidateQueries({ queryKey: ["dashboard"] });
      queryClient.invalidateQueries({ queryKey: ["reports"] });
    },
  });
}
```

type Variables = {
  userId: string
}

app.use('*', cors({
  origin: process.env.FRONTEND_URL ?? 'http://localhost:5173',

// Health check

// better-auth handler — handles /api/auth/*

// Auth middleware for all other /api/* routes

  app.get('*', serveStatic({ path: './apps/web/dist/index.html' }))
}

// Bun native server
export default {
  port: parseInt(process.env.PORT ?? '3000'),
  fetch: app.fetch,
}

export { app }

### 4.4. Route Handler Pattern

```typescript
// apps/server/src/routes/transactions.ts
import { Hono } from "hono";
import { zValidator } from "@hono/zod-validator";
import {
  createTransactionSchema,
  updateTransactionSchema,
  transactionFilterSchema,
} from "@pf/shared";
import * as transactionService from "../services/transactionService";
import type { Env } from "../index";

export const transactionRoutes = new Hono<Env>();

// GET /api/transactions
transactionRoutes.get(
  "/",
  zValidator("query", transactionFilterSchema),
  async (c) => {
    const userId = c.get("userId");
    const filters = c.req.valid("query");
    const result = await transactionService.list(userId, filters);
    return c.json(result);
  },
);

// POST /api/transactions
transactionRoutes.post(
  "/",
  zValidator("json", createTransactionSchema),
  async (c) => {
    const userId = c.get("userId");
    const data = c.req.valid("json");
    const transaction = await transactionService.create(userId, data);
    return c.json({ data: transaction }, 201);
  },
);

// PUT /api/transactions/:id
transactionRoutes.put(
  "/:id",
  zValidator("json", updateTransactionSchema),
  async (c) => {
    const userId = c.get("userId");
    const id = c.req.param("id");
    const data = c.req.valid("json");
    const transaction = await transactionService.update(userId, id, data);
    return c.json({ data: transaction });
  },
);

// DELETE /api/transactions/:id
transactionRoutes.delete("/:id", async (c) => {
  const userId = c.get("userId");
  const id = c.req.param("id");
  await transactionService.remove(userId, id);
  return c.json({ data: null }, 200);
});
```

### 4.5. Service Layer Pattern

Service layer chứa business logic, luôn nhận `userId` làm tham số đầu tiên để đảm bảo data isolation:

```typescript
// apps/server/src/services/transactionService.ts
import { db } from "../db";
import { transactions } from "../db/schema";
import { eq, and, desc, sql, between, like, inArray } from "drizzle-orm";
import { paginate } from "../utils/pagination";

export async function list(
  userId: string,
  filters: {
    page?: number;
    limit?: number;
    type?: "income" | "expense";
    categoryId?: string;
    dateFrom?: string;
    dateTo?: string;
    search?: string;
  },
) {
  const { page = 1, limit = 20 } = filters;

  const conditions = [eq(transactions.userId, userId)];

  if (filters.type) {
    conditions.push(eq(transactions.type, filters.type));
  }
  if (filters.categoryId) {
    conditions.push(eq(transactions.categoryId, filters.categoryId));
  }
  if (filters.dateFrom && filters.dateTo) {
    conditions.push(
      between(transactions.date, filters.dateFrom, filters.dateTo),
    );
  }
  if (filters.search) {
    conditions.push(like(transactions.note, `%${filters.search}%`));
  }

  const where = and(...conditions);

  const [data, total] = await Promise.all([
    db
      .select()
      .from(transactions)
      .where(where)
      .orderBy(desc(transactions.date))
      .limit(limit)
      .offset((page - 1) * limit),
    db
      .select({ count: sql<number>`count(*)` })
      .from(transactions)
      .where(where),
  ]);

  return {
    data,
    meta: {
      page,
      limit,
      total: Number(total[0].count),
      totalPages: Math.ceil(Number(total[0].count) / limit),
    },
  };
}

export async function create(
  userId: string,
  data: {
    amount: number; // integer cents
    type: "income" | "expense";
    categoryId: string;
    walletId: string;
    date: string;
    note?: string;
  },
) {
  await db
    .insert(transactions)
    .values({ ...data, userId });

  // SQLite drizzle: .returning() không khả dụng — query lại nếu cần
  // Hoặc dùng sql helper để lấy last_insert_rowid()
}

export async function update(
  userId: string,
  id: string,
  data: {
    amount?: number; // integer cents
    categoryId?: string;
    walletId?: string;
    date?: string;
    note?: string;
  },
) {
  const result = await db
    .update(transactions)
    .set({ ...data, updatedAt: sql`(datetime('now'))` })
    .where(and(eq(transactions.id, id), eq(transactions.userId, userId)));

  if (result.changes === 0) {
    throw new Error("Transaction not found");
  }
}

export async function remove(userId: string, id: string) {
  const result = await db
    .delete(transactions)
    .where(and(eq(transactions.id, id), eq(transactions.userId, userId)));

  if (result.changes === 0) {
    throw new Error("Transaction not found");
  }
}
```

### 4.6. Error Handling

```typescript
// apps/server/src/middleware/errorHandler.ts
import type { ErrorHandler } from "hono";

export const errorHandler: ErrorHandler = (err, c) => {
  console.error(`[Error] ${err.message}`, err.stack);

  // Zod validation errors
  if (err.name === "ZodError") {
    return c.json(
      {
        error: {
          code: "VALIDATION_ERROR",
          message: "Invalid request data",
          details: JSON.parse(err.message),
        },
      },
      400,
    );
  }

  // Business logic errors
  if (err.message.includes("not found")) {
    return c.json(
      { error: { code: "NOT_FOUND", message: err.message } },
      404,
    );
  }

  // Generic server error
  return c.json(
    {
      error: {
        code: "INTERNAL_ERROR",
        message: "Something went wrong",
      },
    },
    500,
  );
};
```

### 5.2. Drizzle Schema Definitions

```typescript
// apps/server/src/db/schema.ts
import { sqliteTable, text, integer } from 'drizzle-orm/sqlite-core'
import { relations, sql } from 'drizzle-orm'

// ─── Helpers ────────────────────────────────────────────
const id = () => text('id').primaryKey().$defaultFn(() => crypto.randomUUID())
const createdAt = () => text('created_at').notNull().default(sql`(datetime('now'))`)
const updatedAt = () => text('updated_at').notNull().default(sql`(datetime('now'))`)

// ─── User Profiles ──────────────────────────────────────

export const userProfiles = sqliteTable('user_profiles', {
  id: id(),
  authUserId: text('auth_user_id').notNull().unique(), // FK → user (better-auth)
  displayName: text('display_name'),
  currency: text('currency').notNull().default('VND'),
  theme: text('theme').notNull().default('system'), // 'light' | 'dark' | 'system'
  createdAt: createdAt(),
  updatedAt: updatedAt(),
})

// ─── Category Groups ────────────────────────────────────

export const categoryGroups = sqliteTable('category_groups', {
  id: id(),
  userId: text('user_id').notNull().references(() => userProfiles.id, { onDelete: 'cascade' }),
  name: text('name').notNull(),
  type: text('type').notNull(), // 'income' | 'expense'
  color: text('color'),
  sortOrder: integer('sort_order').notNull().default(0),
  isDefault: integer('is_default', { mode: 'boolean' }).notNull().default(false),
  createdAt: createdAt(),
})

// ─── Categories ─────────────────────────────────────────

export const categories = sqliteTable('categories', {
  id: id(),
  userId: text('user_id').notNull().references(() => userProfiles.id, { onDelete: 'cascade' }),
  groupId: text('group_id').notNull().references(() => categoryGroups.id, { onDelete: 'cascade' }),
  name: text('name').notNull(),
  type: text('type').notNull(), // 'income' | 'expense'
  icon: text('icon'),
  color: text('color'),
  isVisible: integer('is_visible', { mode: 'boolean' }).notNull().default(true),
  isDefault: integer('is_default', { mode: 'boolean' }).notNull().default(false),
  sortOrder: integer('sort_order').notNull().default(0),
  createdAt: createdAt(),
})

// ─── Wallets ────────────────────────────────────────────

export const wallets = sqliteTable('wallets', {
  id: id(),
  userId: text('user_id').notNull().references(() => userProfiles.id, { onDelete: 'cascade' }),
  name: text('name').notNull(),
  type: text('type').notNull(), // 'cash' | 'bank' | 'e-wallet'
  initialBalance: integer('initial_balance').notNull().default(0), // stored in cents
  isDefault: integer('is_default', { mode: 'boolean' }).notNull().default(false),
  createdAt: createdAt(),
  updatedAt: updatedAt(),
})

// ─── Transactions ───────────────────────────────────────

export const transactions = sqliteTable('transactions', {
  id: id(),
  userId: text('user_id').notNull().references(() => userProfiles.id, { onDelete: 'cascade' }),
  type: text('type').notNull(), // 'income' | 'expense' | 'transfer'
  amount: integer('amount').notNull(), // stored in cents
  categoryId: text('category_id').notNull().references(() => categories.id),
  walletId: text('wallet_id').notNull().references(() => wallets.id),
  toWalletId: text('to_wallet_id').references(() => wallets.id), // Cho transfer: wallet đích
  date: text('date').notNull(), // YYYY-MM-DD
  note: text('note'),
  isRecurring: integer('is_recurring', { mode: 'boolean' }).notNull().default(false),
  createdAt: createdAt(),
  updatedAt: updatedAt(),
})

// ─── Recurring Templates ────────────────────────────────

export const recurringTemplates = sqliteTable('recurring_templates', {
  id: id(),
  userId: text('user_id').notNull().references(() => userProfiles.id, { onDelete: 'cascade' }),
  type: text('type').notNull(), // 'income' | 'expense' | 'transfer'
  amount: integer('amount').notNull(), // stored in cents
  categoryId: text('category_id').notNull().references(() => categories.id),
  walletId: text('wallet_id').notNull().references(() => wallets.id),
  note: text('note'),
  interval: text('interval').notNull(), // 'weekly' | 'monthly' | 'yearly'
  startDate: text('start_date').notNull(), // YYYY-MM-DD
  endDate: text('end_date'),
  lastGeneratedDate: text('last_generated_date'),
  isActive: integer('is_active', { mode: 'boolean' }).notNull().default(true),
  createdAt: createdAt(),
})

// ─── Budgets ────────────────────────────────────────────

export const budgets = sqliteTable('budgets', {
  id: id(),
  userId: text('user_id').notNull().references(() => userProfiles.id, { onDelete: 'cascade' }),
  month: text('month').notNull(), // YYYY-MM
  totalLimit: integer('total_limit').notNull(), // stored in cents
  createdAt: createdAt(),
  updatedAt: updatedAt(),
})

export const categoryBudgets = sqliteTable('category_budgets', {
  id: id(),
  budgetId: text('budget_id').notNull().references(() => budgets.id, { onDelete: 'cascade' }),
  categoryId: text('category_id').notNull().references(() => categories.id),
  limitAmount: integer('limit_amount').notNull(), // stored in cents
})

// ─── Assets ─────────────────────────────────────────────

export const assets = sqliteTable('assets', {
  id: id(),
  userId: text('user_id').notNull().references(() => userProfiles.id, { onDelete: 'cascade' }),
  type: text('type').notNull(), // 'cash' | 'bank' | 'gold' | 'stock' | 'savings' | 'real_estate' | 'debt'
  name: text('name').notNull(),
  currentValue: integer('current_value').notNull().default(0), // stored in cents
  // Metadata JSON cho field đặc thù theo loại tài sản
  // Gold: { unit, quantity, purchasePrice }
  // Stock: { tickerSymbol, quantity, avgPurchasePrice }
  // Savings: { bankName, principal, interestRate, termMonths, depositDate }
  // Debt: { principal, interestRate, loanDate, termMonths, amountPaid }
  // RealEstate: { estimatedValue }
  metadata: text('metadata'), // JSON string
  note: text('note'),
  createdAt: createdAt(),
  updatedAt: updatedAt(),
})

// ─── Asset History ──────────────────────────────────────

export const assetHistory = sqliteTable('asset_history', {
  id: id(),
  assetId: text('asset_id').notNull().references(() => assets.id, { onDelete: 'cascade' }),
  date: text('date').notNull(), // YYYY-MM-DD
  value: integer('value').notNull(), // stored in cents
  createdAt: createdAt(),
})
```

> **Ghi chú SQLite:** SQLite không có enum type — dùng `text()` với comment ghi rõ giá trị hợp lệ. Boolean dùng `integer({ mode: 'boolean' })`. Monetary values lưu dạng integer cents. Dates lưu dạng text ISO 8601.

### 6.3. Signup Flow

```
1. User điền form đăng ký (email, password, name)
2. Frontend: authClient.signUp.email({ email, password, name })
   → POST /api/auth/sign-up (Hono → better-auth handler)
   → better-auth tạo user record trong bảng `user` + `account`
   ← Set-Cookie: better-auth session cookie
3. Frontend: POST /api/auth/setup-profile { displayName } (gọi sau signUp)
4. Backend:
   a. Tạo record trong user_profiles (authUserId, displayName, currency=VND)
   b. Seed default category_groups + categories (6 nhóm chi, 4 nhóm thu)
   c. Tạo default wallet (Tiền mặt, type=cash, isDefault=true)
5. Redirect → Dashboard
```

### 6.4. Login Flow

```
1. User điền email + password
2. Frontend: authClient.signIn.email({ email, password })
   → POST /api/auth/sign-in (Hono → better-auth handler)
   → better-auth verify credentials
   ← Set-Cookie: better-auth session cookie
3. useSession() tự động cập nhật session state
4. ProtectedRoute cho phép truy cập → Redirect → Dashboard
```

### 6.5. Session Management & ProtectedRoute

better-auth sử dụng cookie-based sessions với auto-refresh. Không cần quản lý JWT tokens.

```typescript
// apps/web/src/components/layout/ProtectedRoute.tsx
import { Navigate, Outlet } from "react-router";
import { authClient } from "../../lib/auth-client";

export function ProtectedRoute() {
  const { data: session, isPending } = authClient.useSession();

  if (isPending) {
    return <div className="flex h-screen items-center justify-center">Loading...</div>;
  }

  if (!session) {
    return <Navigate to="/login" replace />;
  }

  return <Outlet />;
}
```

### 6.6. Backend Verify

Backend dùng `auth.api.getSession({ headers })` inline middleware (xem mục 4.2). Session được verify từ cookie, không cần JWT hay service role key.

### 6.7. Post-Signup Seeding

```typescript
// apps/server/src/utils/seed.ts
import { db } from "../db";
import { userProfiles, categoryGroups, categories, wallets } from "../db/schema";
import { defaultCategories } from "@pf/shared";

export async function seedUserData(authUserId: string, displayName: string) {
  // 1. Tạo user profile
  const profileId = crypto.randomUUID();
  db.insert(userProfiles)
    .values({ id: profileId, authUserId, displayName })
    .run();

  // 2. Seed default categories (từ shared constants)
  for (const group of defaultCategories) {
    const groupId = crypto.randomUUID();
    db.insert(categoryGroups)
      .values({
        id: groupId,
        userId: profileId,
        name: group.name,
        type: group.type,
        color: group.color,
        isDefault: true,
      })
      .run();

    for (const cat of group.categories) {
      db.insert(categories).values({
        userId: profileId,
        groupId,
        name: cat.name,
        type: group.type,
        icon: cat.icon,
        color: cat.color,
        isDefault: true,
      }).run();
    }
  }

  // 3. Tạo default wallet
  db.insert(wallets).values({
    userId: profileId,
    name: "Tiền mặt",
    type: "cash",
    isDefault: true,
  }).run();

  return { id: profileId, authUserId, displayName };
}
```

### 7.6. Date & Currency Conventions

| Quy ước | Chi tiết |
|:---|:---|
| **Date format** | ISO 8601: `"2026-02-28"` (date only, text trong SQLite), `"2026-02-28T10:30:00Z"` (timestamp) |
| **Month format** | `"YYYY-MM"` (ví dụ: `"2026-02"`) — dùng cho budgets |
| **Currency storage** | `integer` — stored as **cents** (ví dụ: 150000₫ → `15000000` cents) |
| **Currency in JSON** | Trả về số nguyên (integer cents), frontend convert: `displayValue = dbValue / 100` |
| **Currency display** | Format trên frontend: `1.234.567 ₫` (Vietnamese locale, dot separator) |
| **Timezone** | Server lưu UTC, frontend convert theo timezone user |

    .int()

  walletId: z.string().uuid().optional(),
  dateFrom: z.string().date().optional(),
  dateTo: z.string().date().optional(),

## 9. Ghi chú Triển khai theo Phase

### Phase 1 — MVP

| Thành phần | Chi tiết |
|:---|:---|
| **DB Tables** | `user_profiles`, `category_groups`, `categories`, `wallets`, `transactions` |
| **API Routes** | `/auth/*`, `/transactions/*`, `/categories/*`, `/category-groups/*`, `/wallets/*`, `/dashboard` |
| **Pages** | LoginPage, RegisterPage, DashboardPage, TransactionsPage, CategoriesPage |
| **Shared Schemas** | auth, transaction, category, wallet schemas + defaultCategories constant |
| **Hooks** | useAuth, useTransactions, useCategories, useWallets, useDashboard |

### Phase 2 — Core Features

| Thành phần | Chi tiết |
|:---|:---|
| **DB Tables** | `recurring_templates`, `budgets`, `category_budgets`, `assets`, `asset_history` |
| **API Routes** | `/budgets/*`, `/assets/*`, `/recurring-templates/*`, `/reports/*`, `/transactions/export` |
| **Pages** | BudgetPage, AssetsPage, ReportsPage |
| **Shared Schemas** | budget, asset, recurringTemplate schemas + assetTypes constant |
| **Hooks** | useBudgets, useAssets, useRecurringTemplates, useReports |

### Phase 3 — Enhancement

| Thành phần | Chi tiết |
|:---|:---|
| **DB Tables** | Không thêm mới |
| **API Routes** | `/settings/*` (profile, backup, restore, delete-all) |
| **Pages** | SettingsPage |
| **Shared Schemas** | settings schemas |
| **Nâng cấp** | Responsive mobile, theme toggle, backup/restore, performance optimization |
