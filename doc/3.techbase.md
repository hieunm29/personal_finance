## 2. Frontend

### 2.1. React 19 + TypeScript

React được chọn vì ecosystem lớn nhất trong các frontend framework, tài liệu phong phú, và TypeScript LSP đã được cấu hình sẵn trong Neovim (LazyVim).

TypeScript đảm bảo type-safety xuyên suốt codebase, giảm lỗi runtime và hỗ trợ refactor an toàn.

### 2.2. Vite 6

Build tool hiện đại, khởi động dev server gần như tức thì nhờ native ES modules. Vite 6 hỗ trợ Bun làm runtime, tận dụng tốc độ của Bun trong quá trình phát triển.

### 2.3. TailwindCSS 4

Utility-first CSS framework, loại bỏ nhu cầu viết CSS riêng cho từng component. TailwindCSS 4 có engine mới (Oxide) nhanh hơn đáng kể. Tailwind LSP đã được cài trong Neovim, hỗ trợ autocomplete class names.

### 2.4. State Management

| Loại state | Công cụ | Mục đích |
|:---|:---|:---|
| **Server state** | TanStack Query v5 | Fetch, cache, sync dữ liệu từ API. Tự động refetch khi focus, retry khi lỗi, invalidate khi mutation |
| **Client state** | Zustand | UI state thuần túy (theme, sidebar, modal). API đơn giản, không boilerplate như Redux |

### 2.5. Biểu đồ (Charts)

Chart.js 4 kết hợp react-chartjs-2 — đã được kiểm chứng trong prototype với các loại biểu đồ: bar, line, doughnut, area. Nhẹ hơn Recharts, đủ mạnh cho 8 loại biểu đồ trong yêu cầu.

### 2.6. Form & Validation

- **react-hook-form**: Quản lý form không re-render toàn bộ, hiệu năng cao.
- **zod**: Schema validation type-safe, chia sẻ schema giữa frontend và backend (cùng monorepo).

### 2.7. Routing

React Router v7 xử lý SPA routing với các trang chính: Dashboard, Giao dịch, Danh mục, Ngân sách, Tài sản, Báo cáo, Cài đặt. Hỗ trợ protected routes cho các trang yêu cầu đăng nhập.

## 3. Backend

### 3.1. Bun Runtime

Bun là JavaScript/TypeScript runtime nhanh, đã được cài sẵn trên máy dev. Ưu điểm chính:
- Bundler, test runner, package manager tích hợp.
- Tương thích hầu hết các package npm.
- Tốc độ khởi động và install nhanh hơn Node.js.

> **Lưu ý:** Development dùng Bun runtime, production trên Vercel dùng Node.js runtime. Hono là multi-runtime framework nên tương thích cả hai mà không cần thay đổi code.

### 3.2. Hono Framework

Web framework siêu nhẹ (~14KB), hỗ trợ multi-runtime (Bun, Node, Deno, Cloudflare Workers). Lý do chọn Hono thay vì ElysiaJS:
- Cộng đồng lớn hơn, tài liệu tốt hơn.
- Middleware ecosystem phong phú (cors, logger, jwt, zod-validator...).
- API giống Express — dễ tiếp cận.

### 3.3. Drizzle ORM

ORM type-safe cho TypeScript, hỗ trợ PostgreSQL qua `postgres` (postgres-js) driver. Ưu điểm:
- Schema định nghĩa bằng TypeScript, tạo type tự động.
- Migration được quản lý bởi `drizzle-kit` (generate + migrate).
- Query builder gần gũi với SQL, không "magic" như Prisma.
- Nhẹ, không cần generate client riêng.

### 3.4. Authentication — Supabase Auth

Sử dụng **Supabase Auth** (`@supabase/supabase-js`) thay vì tự implement JWT + password hashing. Supabase quản lý toàn bộ user, session, và token.

| Thành phần | Vai trò |
|:---|:---|
| **Supabase Auth (client)** | Đăng ký, đăng nhập, quản lý session từ frontend |
| **Supabase Auth (server)** | Verify JWT token từ request, lấy thông tin user |

Luồng auth:
1. Đăng ký/Đăng nhập → Frontend gọi `supabase.auth.signUp()` / `supabase.auth.signInWithPassword()`.
2. Supabase trả về JWT access token + refresh token, tự động quản lý session.
3. Frontend gửi access token trong header `Authorization: Bearer <token>` cho mỗi API request.
4. Backend verify token bằng Supabase client → lấy `user.id` để xử lý business logic.
5. Token hết hạn → Supabase client tự động refresh.

### 3.5. Validation

Zod được dùng chung cho cả frontend và backend. Backend validate mọi request body/query params trước khi xử lý. Schema zod có thể export từ package chung trong monorepo.

## 4. Database

### 4.1. Supabase PostgreSQL — Lý do chọn

| Tiêu chí | Giá trị |
|:---|:---|
| **Database** | PostgreSQL (managed bởi Supabase) |
| **Free tier** | 500MB storage, 2 projects — đủ cho ~10 users |
| **Dashboard** | Supabase Studio — quản lý data, chạy SQL trực tiếp trên web |
| **Backup** | Auto backup hàng ngày (Supabase quản lý) |
| **Auth tích hợp** | Supabase Auth dùng chung database, bảng `auth.users` tự động |
| **Connection** | Connection string qua env var `DATABASE_URL` |

Supabase cung cấp managed PostgreSQL với dashboard quản lý, auto backup, và Auth tích hợp — giảm đáng kể complexity so với self-hosted.

### 4.2. Cấu hình

```typescript
import { drizzle } from "drizzle-orm/postgres-js";
import postgres from "postgres";
import * as schema from "./schema";

const connectionString = process.env.DATABASE_URL!;
const client = postgres(connectionString);
export const db = drizzle(client, { schema });
```

Drizzle config (`drizzle.config.ts`):

```typescript
import { defineConfig } from "drizzle-kit";

export default defineConfig({
  schema: "./src/db/schema.ts",
  out: "./src/db/migrations",
  dialect: "postgresql",
  dbCredentials: {
    url: process.env.DATABASE_URL!,
  },
});
```

## 6. Triển khai (Deployment)

### 6.1. Vercel

Ứng dụng deploy trên Vercel với 2 phần:
- **Frontend**: Vite build → static files, serve bởi Vercel CDN.
- **Backend**: Hono + `@hono/vercel` adapter → Vercel Serverless Functions (Node.js runtime).

### 6.2. Vercel Config

```json
{
  "buildCommand": "bun run build",
  "outputDirectory": "apps/web/dist",
  "rewrites": [
    { "source": "/api/(.*)", "destination": "/api" }
  ]
}
```

Backend entry point cho Vercel (`api/index.ts`):

```typescript
import { handle } from "@hono/vercel";
import app from "../apps/server/src/index";

export default handle(app);
```

### 6.3. Biến môi trường

| Biến | Mô tả | Mặc định |
|:---|:---|:---|
| `NODE_ENV` | Môi trường chạy | `development` |
| `DATABASE_URL` | Supabase PostgreSQL connection string | — (bắt buộc) |
| `SUPABASE_URL` | Supabase project URL | — (bắt buộc) |
| `SUPABASE_ANON_KEY` | Supabase anonymous/public key | — (bắt buộc) |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service role key (chỉ backend) | — (bắt buộc) |
