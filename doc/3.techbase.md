## 2. Frontend

### 2.1. React 19 + TypeScript

React được chọn vì ecosystem lớn nhất trong các frontend framework, tài liệu phong phú, và TypeScript LSP đã được cấu hình sẵn trong Neovim (LazyVim).

TypeScript đảm bảo type-safety xuyên suốt codebase, giảm lỗi runtime và hỗ trợ refactor an toàn.

### 2.2. Vite 6

Build tool hiện đại, khởi động dev server gần như tức thì nhờ native ES modules. Vite 6 hỗ trợ Bun làm runtime, tận dụng tốc độ của Bun trong quá trình phát triển.

### 2.3. TailwindCSS 4

Utility-first CSS framework, loại bỏ nhu cầu viết CSS riêng cho từng component. TailwindCSS 4 có engine mới (Oxide) nhanh hơn đáng kể. Tailwind LSP đã được cài trong Neovim, hỗ trợ autocomplete class names.

### 2.4. State Management

| Loại state | Công cụ | Mục đích |
|:---|:---|:---|
| **Server state** | TanStack Query v5 | Fetch, cache, sync dữ liệu từ API. Tự động refetch khi focus, retry khi lỗi, invalidate khi mutation |
| **Client state** | Zustand | UI state thuần túy (theme, sidebar, modal). API đơn giản, không boilerplate như Redux |

### 2.5. Biểu đồ (Charts)

Chart.js 4 kết hợp react-chartjs-2 — đã được kiểm chứng trong prototype với các loại biểu đồ: bar, line, doughnut, area. Nhẹ hơn Recharts, đủ mạnh cho 8 loại biểu đồ trong yêu cầu.

### 2.6. Form & Validation

- **react-hook-form**: Quản lý form không re-render toàn bộ, hiệu năng cao.
- **zod**: Schema validation type-safe, chia sẻ schema giữa frontend và backend (cùng monorepo).

### 2.7. Routing

React Router v7 xử lý SPA routing với các trang chính: Dashboard, Giao dịch, Danh mục, Ngân sách, Tài sản, Báo cáo, Cài đặt. Hỗ trợ protected routes cho các trang yêu cầu đăng nhập.

## 3. Backend

### 3.1. Bun Runtime

Bun là JavaScript/TypeScript runtime nhanh, đã được cài sẵn trên máy dev. Ưu điểm chính:
- Bundler, test runner, package manager tích hợp.
- Tương thích hầu hết các package npm.
- Tốc độ khởi động và install nhanh hơn Node.js.
- Native SQLite driver (`bun:sqlite`) — không cần package bên ngoài.

> **Lưu ý:** Cả development lẫn production đều chạy trên Bun runtime. Server Hono phục vụ cả API lẫn static files trong production.

### 3.2. Hono Framework

Web framework siêu nhẹ (~14KB), hỗ trợ multi-runtime (Bun, Node, Deno, Cloudflare Workers). Lý do chọn Hono thay vì ElysiaJS:
- Cộng đồng lớn hơn, tài liệu tốt hơn.
- Middleware ecosystem phong phú (cors, logger, zod-validator...).
- API giống Express — dễ tiếp cận.

### 3.3. Drizzle ORM

ORM type-safe cho TypeScript, sử dụng SQLite dialect qua `bun:sqlite` native driver. Ưu điểm:
- Schema định nghĩa bằng TypeScript, tạo type tự động.
- Migration được quản lý bởi `drizzle-kit` (generate + migrate + push).
- Query builder gần gũi với SQL, không "magic" như Prisma.
- Nhẹ, không cần generate client riêng.

### 3.4. Authentication — better-auth

Sử dụng **better-auth** — thư viện auth self-hosted, tích hợp trực tiếp vào Hono server. better-auth quản lý user, session qua cookie-based sessions.

| Thành phần | Vai trò |
|:---|:---|
| **better-auth (server)** | Xử lý đăng ký, đăng nhập, quản lý session, cookie |
| **better-auth/react (client)** | React hooks: `useSession`, `signIn`, `signUp`, `signOut` |

Luồng auth:
1. Đăng ký/Đăng nhập → Frontend gọi `authClient.signUp.email()` / `authClient.signIn.email()`.
2. better-auth tạo user + session, trả cookie `better-auth.session_token`.
3. Frontend gửi cookie tự động nhờ `credentials: 'include'` cho mọi API request.
4. Backend verify session bằng `auth.api.getSession({ headers })` → lấy `user.id`.
5. Session hết hạn → better-auth tự động refresh qua cookie.

better-auth quản lý 4 bảng riêng: `user`, `session`, `account`, `verification`.

### 3.5. Validation

Zod được dùng chung cho cả frontend và backend. Backend validate mọi request body/query params trước khi xử lý. Schema zod có thể export từ package chung trong monorepo.

## 4. Database

### 4.1. SQLite — Lý do chọn

| Tiêu chí | Giá trị |
|:---|:---|
| **Database** | SQLite (local file) |
| **Driver** | `bun:sqlite` — native driver tích hợp trong Bun |
| **Storage** | File `./data/app.db` — portable, dễ backup |
| **Zero-config** | Không cần server riêng, không cần connection string phức tạp |
| **Performance** | WAL mode + PRAGMA tối ưu cho concurrent reads |
| **Auth** | better-auth tạo bảng riêng trong cùng DB |

SQLite phù hợp cho ứng dụng single-user/small-team, không cần managed database service. Dữ liệu lưu trong một file duy nhất, dễ backup và migrate.

### 4.2. Cấu hình

```typescript
// apps/server/src/db/index.ts
import { drizzle } from 'drizzle-orm/bun-sqlite'
import { Database } from 'bun:sqlite'
import * as schema from './schema'
import { mkdirSync } from 'node:fs'
import { dirname, resolve } from 'node:path'

const rawPath = process.env.DATABASE_URL?.replace('file:', '') || './data/app.db'
const dbPath = resolve(rawPath)
mkdirSync(dirname(dbPath), { recursive: true })

const sqlite = new Database(dbPath)

// Optimize SQLite
sqlite.run('PRAGMA journal_mode = WAL')
sqlite.run('PRAGMA synchronous = NORMAL')
sqlite.run('PRAGMA foreign_keys = ON')

export const db = drizzle(sqlite, { schema })
```

Drizzle config (`drizzle.config.ts`):

```typescript
import { defineConfig } from 'drizzle-kit'

export default defineConfig({
  dialect: 'sqlite',
  schema: './src/db/schema.ts',
  out: './src/db/migrations',
  dbCredentials: {
    url: process.env.DATABASE_URL?.replace('file:', '') || '../../data/app.db',
  },
})
```

### 4.3. SQLite vs PostgreSQL — Khác biệt chính

| PostgreSQL | SQLite |
|:---|:---|
| `pgTable` | `sqliteTable` |
| `pgEnum(...)` | `text()` + TypeScript union types |
| `uuid().defaultRandom()` | `text().$defaultFn(() => crypto.randomUUID())` |
| `timestamp({ withTimezone })` | `text().default(sql\`(datetime('now'))\`)` |
| `numeric(18, 2)` | `integer()` — lưu **cents** (tránh floating point) |
| `boolean` | `integer({ mode: 'boolean' })` |
| `date` | `text()` — format `YYYY-MM-DD` |

## 6. Triển khai (Deployment)

### 6.1. Docker Compose

Ứng dụng deploy bằng Docker Compose với single container:
- **Frontend**: Vite build → static files, serve bởi Hono (`hono/bun` serveStatic).
- **Backend**: Hono 4 chạy trên Bun native — phục vụ cả API (`/api/*`) lẫn static files.

### 6.2. Dockerfile (Multi-stage)

```dockerfile
# Stage 1: Install deps
FROM oven/bun:1.3-alpine AS deps
WORKDIR /app
COPY package.json bun.lock bunfig.toml ./
COPY apps/server/package.json apps/server/
COPY apps/web/package.json apps/web/
COPY packages/shared/package.json packages/shared/
RUN bun install --frozen-lockfile

# Stage 2: Build frontend
FROM deps AS build
COPY . .
RUN bun run build

# Stage 3: Production
FROM oven/bun:1.3-alpine AS prod
WORKDIR /app
COPY --from=deps /app/node_modules node_modules
COPY --from=deps /app/apps/server/node_modules apps/server/node_modules
COPY --from=deps /app/packages/shared/node_modules packages/shared/node_modules
COPY apps/server apps/server
COPY packages/shared packages/shared
COPY --from=build /app/apps/web/dist apps/web/dist
RUN mkdir -p data
ENV NODE_ENV=production
EXPOSE 3000
CMD ["bun", "run", "apps/server/src/index.ts"]
```

### 6.3. Biến môi trường

| Biến | Mô tả | Mặc định |
|:---|:---|:---|
| `NODE_ENV` | Môi trường chạy | `development` |
| `PORT` | Port server | `3000` |
| `DATABASE_URL` | SQLite file path | `file:./data/app.db` |
| `BETTER_AUTH_SECRET` | Secret key cho better-auth (min 32 chars) | — (bắt buộc) |
| `BETTER_AUTH_URL` | Base URL của server | `http://localhost:3000` |
| `FRONTEND_URL` | URL frontend (CORS origin) | `http://localhost:5173` |
| `VITE_API_URL` | API URL cho frontend | `http://localhost:3000` |
