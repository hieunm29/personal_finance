## 2. Frontend

### 2.1. React 19 + TypeScript

React được chọn vì ecosystem lớn nhất trong các frontend framework, tài liệu phong phú, và TypeScript LSP đã được cấu hình sẵn trong Neovim (LazyVim).

TypeScript đảm bảo type-safety xuyên suốt codebase, giảm lỗi runtime và hỗ trợ refactor an toàn.

### 2.2. Vite 6

Build tool hiện đại, khởi động dev server gần như tức thì nhờ native ES modules. Vite 6 hỗ trợ Bun làm runtime, tận dụng tốc độ của Bun trong quá trình phát triển.

### 2.3. TailwindCSS 4

Utility-first CSS framework, loại bỏ nhu cầu viết CSS riêng cho từng component. TailwindCSS 4 có engine mới (Oxide) nhanh hơn đáng kể. Tailwind LSP đã được cài trong Neovim, hỗ trợ autocomplete class names.

### 2.4. State Management

| Loại state | Công cụ | Mục đích |
|:---|:---|:---|
| **Server state** | TanStack Query v5 | Fetch, cache, sync dữ liệu từ API. Tự động refetch khi focus, retry khi lỗi, invalidate khi mutation |
| **Client state** | Zustand | UI state thuần túy (theme, sidebar, modal). API đơn giản, không boilerplate như Redux |

### 2.5. Biểu đồ (Charts)

Chart.js 4 kết hợp react-chartjs-2 — đã được kiểm chứng trong prototype với các loại biểu đồ: bar, line, doughnut, area. Nhẹ hơn Recharts, đủ mạnh cho 8 loại biểu đồ trong yêu cầu.

### 2.6. Form & Validation

- **react-hook-form**: Quản lý form không re-render toàn bộ, hiệu năng cao.
- **zod**: Schema validation type-safe, chia sẻ schema giữa frontend và backend (cùng monorepo).

### 2.7. Routing

React Router v7 xử lý SPA routing với các trang chính: Dashboard, Giao dịch, Danh mục, Ngân sách, Tài sản, Báo cáo, Cài đặt. Hỗ trợ protected routes cho các trang yêu cầu đăng nhập.

## 3. Backend

### 3.1. Bun Runtime

Bun là JavaScript/TypeScript runtime nhanh, đã được cài sẵn trên máy dev. Ưu điểm chính:
- Tích hợp sẵn SQLite driver (`bun:sqlite`) — không cần package bên ngoài.
- Bundler, test runner, package manager tích hợp.
- Tương thích hầu hết các package npm.

### 3.2. Hono Framework

Web framework siêu nhẹ (~14KB), hỗ trợ multi-runtime (Bun, Node, Deno, Cloudflare Workers). Lý do chọn Hono thay vì ElysiaJS:
- Cộng đồng lớn hơn, tài liệu tốt hơn.
- Middleware ecosystem phong phú (cors, logger, jwt, zod-validator...).
- API giống Express — dễ tiếp cận.

### 3.3. Drizzle ORM

ORM type-safe cho TypeScript, hỗ trợ SQLite tốt. Ưu điểm:
- Schema định nghĩa bằng TypeScript, tạo type tự động.
- Migration được quản lý bởi `drizzle-kit` (generate + migrate).
- Query builder gần gũi với SQL, không "magic" như Prisma.
- Nhẹ, không cần generate client riêng.

### 3.4. Authentication

| Thư viện | Vai trò |
|:---|:---|
| **jose** | Tạo và xác minh JWT tokens (access token + refresh token) |
| **bcryptjs** | Hash mật khẩu người dùng, pure JS (không cần native bindings) |

Luồng auth:
1. Đăng ký → hash password bằng bcryptjs → lưu vào DB.
2. Đăng nhập → so sánh hash → tạo access token (15 phút) + refresh token (7 ngày).
3. Mỗi request → middleware xác minh access token.
4. Token hết hạn → dùng refresh token để lấy access token mới.

### 3.5. Validation

Zod được dùng chung cho cả frontend và backend. Backend validate mọi request body/query params trước khi xử lý. Schema zod có thể export từ package chung trong monorepo.

## 4. Database

### 4.1. SQLite — Lý do chọn

| Tiêu chí | SQLite | PostgreSQL |
|:---|:---|:---|
| **Cài đặt** | Không cần — tích hợp trong Bun | Cần cài riêng, cấu hình connection |
| **Vận hành** | Zero admin, không có process riêng | Cần quản lý process, user, permissions |
| **Backup** | Copy file `.db` | pg_dump hoặc logical replication |
| **Hiệu năng ~10 users** | Quá đủ, read rất nhanh | Overkill cho quy mô nhỏ |
| **Self-hosted** | Đơn giản nhất | Thêm container riêng |
| **WAL mode** | Hỗ trợ concurrent reads | — |

SQLite hoàn hảo cho ứng dụng ~10 người dùng, self-hosted. File database nằm trong volume Docker, backup bằng cách copy file.

### 4.2. Cấu hình

```typescript
import { Database } from "bun:sqlite";

const db = new Database("data/finance.db", {
  create: true,
  strict: true,
});

// Bật WAL mode để hỗ trợ concurrent reads
db.exec("PRAGMA journal_mode = WAL");
db.exec("PRAGMA foreign_keys = ON");
```

## 6. Triển khai (Deployment)

### 6.1. Docker

Multi-stage build để giữ image nhỏ:

```dockerfile
# Build stage
FROM oven/bun:1 AS builder
WORKDIR /app
COPY . .
RUN bun install --frozen-lockfile
RUN bun run build

# Production stage
FROM oven/bun:1-slim
WORKDIR /app
COPY --from=builder /app/apps/server/dist ./server
COPY --from=builder /app/apps/web/dist ./public
EXPOSE 3000
CMD ["bun", "run", "server/index.js"]
```

### 6.2. Docker Compose

```yaml
version: "3.8"
services:
  app:
    build: .
    ports:
      - "${PORT:-3000}:3000"
    volumes:
      - ./data:/app/data        # SQLite database persist
    environment:
      - NODE_ENV=production
      - JWT_SECRET=${JWT_SECRET}
      - JWT_REFRESH_SECRET=${JWT_REFRESH_SECRET}
      - DATABASE_PATH=/app/data/finance.db
    restart: unless-stopped
```

### 6.3. Biến môi trường

| Biến | Mô tả | Mặc định |
|:---|:---|:---|
| `PORT` | Port server lắng nghe | `3000` |
| `NODE_ENV` | Môi trường chạy | `development` |
| `JWT_SECRET` | Secret key cho access token | — (bắt buộc) |
| `JWT_REFRESH_SECRET` | Secret key cho refresh token | — (bắt buộc) |
| `DATABASE_PATH` | Đường dẫn file SQLite | `data/finance.db` |
