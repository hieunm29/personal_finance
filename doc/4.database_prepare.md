# Database Prepare — Schema & Dữ liệu Test

## Mục đích

File này mô tả cấu trúc database **SQLite** và cách tạo dữ liệu test. Schema được quản lý bởi **Drizzle ORM** (SQLite dialect, `bun:sqlite` driver).

### Cách sử dụng

1. Chạy `bun run db:generate` để tạo migration từ Drizzle schema
2. Chạy `bun run db:migrate` để apply migration (hoặc `bunx drizzle-kit push` cho dev)
3. Chạy `bun run db:seed` để insert dữ liệu test

> **Lưu ý:**
> - Database file: `./data/app.db` (tự tạo khi server khởi động)
> - better-auth tự quản lý 4 bảng riêng: `user`, `session`, `account`, `verification`
> - App tables dùng Drizzle schema riêng, `user_profiles.authUserId` FK tới `user.id` (bảng better-auth)

## Phần 1 — Drizzle Schema (SQLite)

### 1.1. Khác biệt so với PostgreSQL (tham khảo)

| PostgreSQL | SQLite (hiện tại) |
|:---|:---|
| `pgTable`, `pgEnum` | `sqliteTable`, không có enum — dùng `text()` + TypeScript types |
| `uuid().defaultRandom()` | `text().$defaultFn(() => crypto.randomUUID())` |
| `timestamp({ withTimezone })` | `text().default(sql\`(datetime('now'))\`)` |
| `numeric(18, 2)` | `integer()` — lưu **cents** (tránh floating point) |
| `boolean` | `integer({ mode: 'boolean' })` |
| `date` | `text()` — format `YYYY-MM-DD` |

### 1.2. Schema (10 bảng)

Schema đầy đủ nằm tại `apps/server/src/db/schema.ts`. Tóm tắt:

| Bảng | Mô tả | Monetary fields (cents) |
|:---|:---|:---|
| `user_profiles` | Profile người dùng, FK tới better-auth `user.id` | — |
| `category_groups` | Nhóm danh mục (thu/chi) | — |
| `categories` | Danh mục con | — |
| `wallets` | Ví tiền | `initial_balance` |
| `transactions` | Giao dịch | `amount` |
| `recurring_templates` | Template giao dịch định kỳ | `amount` |
| `budgets` | Ngân sách tháng | `total_limit` |
| `category_budgets` | Ngân sách theo danh mục | `limit_amount` |
| `assets` | Tài sản | `current_value` |
| `asset_history` | Lịch sử giá trị tài sản | `value` |

### 1.3. Quy ước lưu tiền tệ (cents)

Tất cả monetary values lưu dưới dạng **integer cents** để tránh floating point errors:
- `1,500,000 VND` → lưu `150000000` (1.5 triệu × 100)
- `$15.50` → lưu `1550`
- Frontend chuyển đổi: `displayValue = dbValue / 100`

### Tham khảo SQL (tương đương seed script)

Dưới đây là SQL reference cho các bảng chính (dùng cho debugging hoặc chạy trực tiếp trong SQLite CLI):

```sql
-- ═══════════════════════════════════════════════════════════
-- TABLES (SQLite — theo thứ tự FK dependencies)
-- Lưu ý: Schema thực tế quản lý bởi Drizzle ORM
-- ═══════════════════════════════════════════════════════════

-- 1. user_profiles
CREATE TABLE user_profiles (
  id              TEXT PRIMARY KEY,
  auth_user_id    TEXT NOT NULL UNIQUE,  -- FK → better-auth user.id
  display_name    TEXT,
  currency        TEXT NOT NULL DEFAULT 'VND',
  theme           TEXT NOT NULL DEFAULT 'system',  -- 'light' | 'dark' | 'system'
  created_at      TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at      TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 2. category_groups
CREATE TABLE category_groups (
  id              TEXT PRIMARY KEY,
  user_id         TEXT NOT NULL REFERENCES user_profiles(id) ON DELETE CASCADE,
  name            TEXT NOT NULL,
  type            TEXT NOT NULL,  -- 'income' | 'expense'
  color           TEXT,
  sort_order      INTEGER NOT NULL DEFAULT 0,
  is_default      INTEGER NOT NULL DEFAULT 0,  -- boolean
  created_at      TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 3. categories
CREATE TABLE categories (
  id              TEXT PRIMARY KEY,
  user_id         TEXT NOT NULL REFERENCES user_profiles(id) ON DELETE CASCADE,
  group_id        TEXT NOT NULL REFERENCES category_groups(id) ON DELETE CASCADE,
  name            TEXT NOT NULL,
  type            TEXT NOT NULL,  -- 'income' | 'expense'
  icon            TEXT,
  color           TEXT,
  is_visible      INTEGER NOT NULL DEFAULT 1,  -- boolean
  is_default      INTEGER NOT NULL DEFAULT 0,
  sort_order      INTEGER NOT NULL DEFAULT 0,
  created_at      TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 4. wallets
CREATE TABLE wallets (
  id              TEXT PRIMARY KEY,
  user_id         TEXT NOT NULL REFERENCES user_profiles(id) ON DELETE CASCADE,
  name            TEXT NOT NULL,
  type            TEXT NOT NULL,  -- 'cash' | 'bank' | 'e-wallet'
  initial_balance INTEGER NOT NULL DEFAULT 0,  -- cents
  is_default      INTEGER NOT NULL DEFAULT 0,
  created_at      TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at      TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 5. transactions
CREATE TABLE transactions (
  id              TEXT PRIMARY KEY,
  user_id         TEXT NOT NULL REFERENCES user_profiles(id) ON DELETE CASCADE,
  type            TEXT NOT NULL,  -- 'income' | 'expense' | 'transfer'
  amount          INTEGER NOT NULL,  -- cents
  category_id     TEXT NOT NULL REFERENCES categories(id),
  wallet_id       TEXT NOT NULL REFERENCES wallets(id),
  to_wallet_id    TEXT REFERENCES wallets(id),
  date            TEXT NOT NULL,  -- YYYY-MM-DD
  note            TEXT,
  is_recurring    INTEGER NOT NULL DEFAULT 0,
  created_at      TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at      TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 6. recurring_templates
CREATE TABLE recurring_templates (
  id                  TEXT PRIMARY KEY,
  user_id             TEXT NOT NULL REFERENCES user_profiles(id) ON DELETE CASCADE,
  type                TEXT NOT NULL,
  amount              INTEGER NOT NULL,  -- cents
  category_id         TEXT NOT NULL REFERENCES categories(id),
  wallet_id           TEXT NOT NULL REFERENCES wallets(id),
  note                TEXT,
  interval            TEXT NOT NULL,  -- 'weekly' | 'monthly' | 'yearly'
  start_date          TEXT NOT NULL,
  end_date            TEXT,
  last_generated_date TEXT,
  is_active           INTEGER NOT NULL DEFAULT 1,
  created_at          TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 7. budgets
CREATE TABLE budgets (
  id              TEXT PRIMARY KEY,
  user_id         TEXT NOT NULL REFERENCES user_profiles(id) ON DELETE CASCADE,
  month           TEXT NOT NULL,  -- 'YYYY-MM'
  total_limit     INTEGER NOT NULL,  -- cents
  created_at      TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at      TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 8. category_budgets
CREATE TABLE category_budgets (
  id              TEXT PRIMARY KEY,
  budget_id       TEXT NOT NULL REFERENCES budgets(id) ON DELETE CASCADE,
  category_id     TEXT NOT NULL REFERENCES categories(id),
  limit_amount    INTEGER NOT NULL  -- cents
);

-- 9. assets
CREATE TABLE assets (
  id              TEXT PRIMARY KEY,
  user_id         TEXT NOT NULL REFERENCES user_profiles(id) ON DELETE CASCADE,
  type            TEXT NOT NULL,  -- 'cash' | 'bank' | 'gold' | 'stock' | 'savings' | 'real_estate' | 'debt'
  name            TEXT NOT NULL,
  current_value   INTEGER NOT NULL DEFAULT 0,  -- cents
  metadata        TEXT,  -- JSON string
  note            TEXT,
  created_at      TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at      TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 10. asset_history
CREATE TABLE asset_history (
  id              TEXT PRIMARY KEY,
  asset_id        TEXT NOT NULL REFERENCES assets(id) ON DELETE CASCADE,
  date            TEXT NOT NULL,  -- YYYY-MM-DD
  value           INTEGER NOT NULL,  -- cents
  created_at      TEXT NOT NULL DEFAULT (datetime('now'))
);
```

### PRAGMA Settings

```sql
PRAGMA journal_mode = WAL;       -- Write-Ahead Logging cho concurrent reads
PRAGMA synchronous = NORMAL;     -- Balance giữa safety và performance
PRAGMA foreign_keys = ON;        -- Enforce FK constraints
```

### Quick Verification

```sql
-- Đếm records mỗi bảng
SELECT 'user_profiles' AS tbl, COUNT(*) FROM user_profiles
UNION ALL SELECT 'category_groups', COUNT(*) FROM category_groups
UNION ALL SELECT 'categories', COUNT(*) FROM categories
UNION ALL SELECT 'wallets', COUNT(*) FROM wallets
UNION ALL SELECT 'transactions', COUNT(*) FROM transactions;

-- Kiểm tra FK integrity
SELECT t.id, t.note, c.name AS category, w.name AS wallet
FROM transactions t
JOIN categories c ON t.category_id = c.id
JOIN wallets w ON t.wallet_id = w.id
ORDER BY t.date;
```
